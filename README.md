Задание за първи проект по предметите ООП, РС, БД, ВОТ

Екип: Ескимосите

(1) Тема
Уеб приложение за филми (Movie Explorer App) – платформа за разглеждане на актуални, популярни и търсени филми чрез външно API (напр. TMDb).

(2) Кратко описание
Проектът представлява full-stack уеб приложение, което интегрира външно movie API и позволява на потребителите да:
разглеждат trending и popular филми
търсят по ключова дума
филтрират по жанр
виждат детайлна информация за филм
създават акаунт и запазват любими филми
Приложението ще обработва външни API заявки през собствен backend слой, който ще управлява бизнес логиката, кеширане, обработка на грешки и съхранение на потребителски данни.

(3) Цел на проекта
Целта на проекта е да демонстрира:
Работа с външни REST API
Обработка на асинхронни заявки
Архитектура Client → Server → External API
Работа с база данни
Аутентикация и авторизация
Deployment на реална среда

Проектът цели да покаже разбиране на реални production практики

(4) Функционалности (User Stories)
Публична част
Като потребител искам да виждам популярни филми.
Като потребител искам да търся филми по име.
Като потребител искам да филтрирам по жанр.
Като потребител искам да виждам подробности за филм
Като потребител искам да виждам препоръчани филми на база жанр.

Регистриран потребител
Като потребител искам да създам акаунт.
Като потребител искам да влизам в профила си.
Като потребител искам да добавям филми в "Любими".
Като потребител искам да премахвам филми от "Любими".
Като потребител искам да виждам гледани и негледани филми.

(5) Tech Stack 

Frontend: Next + Tailwind CSS
Защо:
Компонентна архитектура
Лесен state management
Голямо търсене на пазара
Бърз development
Подходящ за SPA приложения

Backend: Node.js
Защо:
Лек и бърз за разработка
Същият език като frontend (JavaScript)
Много подходящ за REST API
Лесна интеграция с външни API

База данни:  PostgreSQL
Защо:
Релационна структура
Подходяща за user → favorites → movies връзки
Production-ready

Архитектура:
Frontend → Backend → External Movie API
Backend → PostgreSQL

(6) Контейнеризация
Проектът ще бъде контейнеризиран чрез Docker, като всички услуги ще бъдат управлявани чрез Docker Compose.
GitHub Action Pipeline за build и push на имиджи в DockerHub.
Database с клъстер, архитектура primary-replica.
Всеки основен компонент ще бъде изолиран в собствен контейнер.
